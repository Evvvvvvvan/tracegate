# TraceGate

一个即插即用的前端行为风控脚本。

不改业务接入方式，不要求主动调用方法。页面只要引入一段 JS，就会自动开始采集访问行为，根据停留时长、鼠标移动轨迹、点击、滚动、操作节奏这些信息做首轮评分。分数高，直接放行；分数低，进入验证，再结合滑块拖动和滚动行为做二次评分。

这个项目想解决的事情很简单：

不是一上来就打断所有人做验证，而是先看一眼访问行为，能自然放行的先放行，只有风险高的时候再补一道验证。

---

## 为什么做这个

很多站点的人机验证方案都比较重：

- 一打开页面就弹验证
- 所有用户一视同仁
- 接入时还要在业务代码里手动调用
- 只靠一个滑块或者一个 token 做判断

这样做的问题也很明显：

- 真用户被打扰
- 接入麻烦
- 前端和业务耦合高
- 风控判断维度太少

TraceGate 走的是另一条路：

先做一层轻量行为识别，把大部分正常访问直接放过去；只有在行为样本不足、轨迹异常、操作节奏不自然的时候，才进入验证流程。

---

## 特点

- **即插即用**
  - 只需要引入 JS
  - 不需要业务侧主动调用任何方法

- **先识别，后验证**
  - 先做首轮行为评分
  - 低分才触发验证

- **双阶段评分**
  - 首轮：停留、轨迹、点击、滚动、速度波动、方向变化
  - 二次：滑块拖动轨迹 + 滚动阅读节奏

- **对正常用户更友好**
  - 行为自然的访问，大多数情况下不会被打断

- **支持服务端联动**
  - 可选上报状态到后端
  - 方便和登录、搜索、发帖、下单、接口限流这些流程打通

- **会话内自动复用结果**
  - 验证通过后，一段时间内不重复打扰

---

## 适合用在什么场景

- 登录页
- 注册页
- 搜索页
- 活动页
- 内容详情页
- 评论、发帖、提交表单前
- 接口请求前的前端风控预判

## 怎么工作

### 1. 页面加载后自动进入观察状态

脚本会自动监听这些行为：

- 停留时长
- 鼠标移动轨迹
- 点击
- 键盘输入
- 页面滚动
- 速度波动
- 方向变化
- 异常跳点
- 可信事件判断

### 2. 首轮评分

在一个短观察窗口内，脚本会给当前访问打一轮分。

- 分数高于阈值：直接放行
- 分数低于阈值：弹出验证层

### 3. 二次验证

验证层不是只看“有没有拖到终点”，还会结合：

- 拖动时长
- 拖动采样点数量
- 拖动过程中的自然回拉
- 滚动阅读节奏
- 整体停留时长
- 是否存在非可信事件

### 4. 会话放行

验证通过后，会在当前会话内记录一次通过状态。后续访问在信任时间内不重复触发验证。

## 快速开始

把脚本放到项目里，例如：

```html
<script>
  window.__ANTI_BOT_CONFIG__ = {
    reportUrl: '/api/anti-bot/report',
    primaryPassScore: 60,
    secondaryPassScore: 55,
    observationWindowMs: 6000,
    sessionTrustMs: 1800000
  };
</script>
<script src="/static/js/anti-bot-guard.js"></script>
